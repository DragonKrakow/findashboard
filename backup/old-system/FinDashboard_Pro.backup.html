AI Assistant (Browser-side Pattern — Reference Only, Disabled)
⚠ Security notice: This section demonstrates a pattern where an AI model is called directly from the browser using a user-supplied API key. This approach is not safe for production because:
  • The key is visible in browser memory, network inspector, and JS source.
  • Any visitor or browser extension could exfiltrate it.
  • There is no rate-limiting or spending cap you can enforce client-side.

Production alternative (used in index.html): Run inference in a GitHub Actions workflow that reads the key from GitHub Secrets. Commit only the resulting JSON to the repository. GitHub Pages serves the static JSON — no key ever reaches the browser.

If you need real-time AI from a static site: Proxy requests through a serverless function (Cloudflare Workers, Vercel Edge Functions, AWS Lambda) that holds the key server-side. The browser sends only the user question; the function appends the key before forwarding to the AI API.
